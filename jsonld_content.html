JSON-LD: 웹 개발자를 위한 RDF

JSON-LD란?

JSON-LD (JSON for Linking Data)는 JSON 형식으로 Linked Data를 표현하는 방법입니다.

기존 JSON에 @context라는 특별한 키를 추가하여 시맨틱 의미를 부여합니다. 웹 개발자에게 가장 친숙한 RDF 직렬화 형식입니다.

  • 개발: 2010년 Manu Sporny 등
  • 표준화: 2014년 W3C 권고안, 2020년 JSON-LD 1.1
  • 특징: JSON과 100% 호환, 웹 개발자 친화적
  • 활용: Schema.org, Google 검색, SEO, 웹 API

1. 왜 JSON-LD인가?

1.1 웹 개발자의 고민

RDF는 어렵다?

많은 웹 개발자들이 RDF를 접하면서 겪는 어려움:

  • Turtle, XML/RDF는 낯선 문법
  • 기존 웹 개발 도구와 호환되지 않음
  • JavaScript에서 직접 다루기 어려움
  • 학습 곡선이 가파름

1.2 JSON-LD의 해결책

JSON으로 RDF를 표현하자!

JSON-LD는 웹 개발자들이 이미 알고 있는 JSON 문법을 그대로 사용합니다:

  • 기존 JSON 파서로 그대로 읽을 수 있음
  • JavaScript에서 바로 사용 가능
  • 웹 API, REST API와 자연스럽게 통합
  • 학습 비용이 낮음

2. JSON-LD 기본 문법

2.1 일반 JSON vs JSON-LD

일반 JSON

{ "name": "Alice", "age": 30, "email": "alice@example.org" }

문제: "name"이 무엇을 의미하는지 컴퓨터가 알 수 없습니다.

JSON-LD

{ "@context": "https://schema.org", "@type": "Person", "name": "Alice", "age": 30, "email": "alice@example.org" }

@context를 추가하여 "name"이 Schema.org의 name 속성임을 명시합니다.

2.2 @context: 의미를 정의하는 맥락

@context란?

@context는 JSON 키(key)에 시맨틱 의미를 부여하는 사전(dictionary)입니다.

각 속성이 어떤 온톨로지의 어떤 용어인지 정의합니다.

{ "@context": { "name": "http://xmlns.com/foaf/0.1/name", "age": "http://xmlns.com/foaf/0.1/age", "email": "http://xmlns.com/foaf/0.1/mbox" }, "name": "Alice", "age": 30, "email": "alice@example.org" }

외부 @context 참조

매번 @context를 작성하는 대신, 공개된 @context를 참조할 수 있습니다:

{ "@context": "https://schema.org", "@type": "Person", "name": "Alice" }

Schema.org의 @context가 자동으로 적용됩니다.

2.3 @id: 리소스 식별자

@id로 리소스에 URI 부여

@id는 RDF의 주어(Subject)에 해당하는 URI를 지정합니다.

{ "@context": "https://schema.org", "@id": "http://example.org/person/alice", "@type": "Person", "name": "Alice" }

2.4 @type: 타입 지정

@type으로 클래스 정의

@type은 리소스의 클래스(rdf:type)를 지정합니다.

{ "@context": "https://schema.org", "@type": "Book", "name": "도서관 정보학 입문", "author": "김철수" }

이 객체가 Schema.org의 Book 타입임을 명시합니다.

2.5 @graph: 여러 리소스 그룹화

@graph로 여러 Triple 묶기

@graph는 여러 리소스를 하나의 JSON 문서에 담을 때 사용합니다.

{ "@context": "https://schema.org", "@graph": [ { "@id": "http://example.org/person/alice", "@type": "Person", "name": "Alice" }, { "@id": "http://example.org/person/bob", "@type": "Person", "name": "Bob" } ] }

3. Schema.org 구조화 데이터

Schema.org란?

Google, Microsoft, Yahoo가 공동 개발한 구조화 데이터 표준입니다.

JSON-LD로 웹페이지에 메타데이터를 추가하면 검색엔진이 더 잘 이해합니다.

3.1 웹사이트에 JSON-LD 추가

HTML에 JSON-LD 삽입

<!-- HTML 문서의 <head> 또는 <body>에 추가 --> <script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Organization", "name": "서울대학교 중앙도서관", "url": "https://library.snu.ac.kr", "logo": "https://library.snu.ac.kr/logo.png", "contactPoint": { "@type": "ContactPoint", "telephone": "+82-2-880-5114", "contactType": "customer service" } } </script>

3.2 책 정보 구조화

Schema.org Book 예시

{ "@context": "https://schema.org", "@type": "Book", "name": "도서관 정보학 입문", "author": { "@type": "Person", "name": "김철수" }, "publisher": { "@type": "Organization", "name": "한국출판사" }, "datePublished": "2023-03-15", "isbn": "978-89-1234-567-8", "numberOfPages": 350, "inLanguage": "ko" }

3.3 이벤트 정보 구조화

도서관 행사 정보

{ "@context": "https://schema.org", "@type": "Event", "name": "정보 리터러시 특강", "description": "대학생을 위한 정보 검색 및 활용 교육", "startDate": "2024-04-15T14:00:00+09:00", "endDate": "2024-04-15T16:00:00+09:00", "location": { "@type": "Place", "name": "중앙도서관 세미나실", "address": { "@type": "PostalAddress", "addressLocality": "서울", "addressCountry": "KR" } }, "organizer": { "@type": "Organization", "name": "중앙도서관" } }

4. JSON vs JSON-LD 비교

일반 JSON

{ "title": "Python 입문", "author": "홍길동", "year": 2023 }
  • 단순한 데이터 교환
  • 의미는 문서화에 의존
  • 기계가 의미를 이해 못함

JSON-LD

{ "@context": "https://schema.org", "@type": "Book", "name": "Python 입문", "author": { "@type": "Person", "name": "홍길동" }, "datePublished": "2023" }
  • 시맨틱 데이터 교환
  • 의미가 명확히 정의됨
  • 기계가 의미를 이해함

5. 도서관에서의 JSON-LD 활용

5.1 BIBFRAME을 JSON-LD로

BIBFRAME Work 표현

{ "@context": "http://bibframe.org/context.json", "@id": "http://example.org/work/123", "@type": "bf:Work", "bf:title": { "@type": "bf:Title", "bf:mainTitle": "시맨틱 웹 기술" }, "bf:contribution": { "@type": "bf:Contribution", "bf:agent": { "@type": "bf:Person", "bf:name": "이순신" }, "bf:role": "author" }, "bf:subject": [ "시맨틱 웹", "RDF", "온톨로지" ] }

5.2 도서관 컬렉션 API

JSON-LD API 응답

// GET /api/books/12345 { "@context": "https://schema.org", "@id": "http://library.example.org/books/12345", "@type": "Book", "name": "Linked Data 실무", "author": { "@id": "http://library.example.org/authors/456", "@type": "Person", "name": "박지성" }, "isbn": "978-89-9999-999-9", "numberOfPages": 420, "offers": { "@type": "Offer", "availability": "https://schema.org/InStock", "inventoryLevel": 3 } }

이 JSON-LD 응답은 일반 JSON처럼 사용하면서도 RDF로 자동 변환 가능합니다.

6. JSON-LD의 장단점

장점 단점
웹 개발자에게 친숙한 JSON 문법 파일 크기가 다른 형식보다 클 수 있음
JavaScript에서 바로 사용 가능 @context 이해가 필요함
기존 JSON API와 호환 복잡한 구조는 표현이 장황함
웹 표준 (W3C 권고안) 완전한 RDF 기능 일부 제한
Schema.org와 완벽 통합 대용량 데이터엔 N-Triples가 효율적
SEO 최적화에 유리 사람이 직접 읽기엔 Turtle이 더 나음

7. 실무 도구 및 라이브러리

JSON-LD 처리 도구

  • jsonld.js: JavaScript용 JSON-LD 프로세서
  • PyLD: Python용 JSON-LD 라이브러리
  • JSON-LD Playground: 온라인 테스트 도구 (json-ld.org/playground)
  • Google Rich Results Test: Schema.org JSON-LD 검증
  • Apache Jena: JSON-LD를 다른 RDF 형식으로 변환

Python 예시

from pyld import jsonld import json # JSON-LD 문서 doc = { "@context": "https://schema.org", "@type": "Person", "name": "Alice" } # N-Quads로 변환 nquads = jsonld.to_rdf(doc, {'format': 'application/n-quads'}) print(nquads) # 확장 (expand) expanded = jsonld.expand(doc) print(json.dumps(expanded, indent=2))

8. JSON-LD 변환

8.1 JSON-LD → Turtle

JSON-LD

{ "@context": "http://xmlns.com/foaf/0.1/", "@id": "http://example.org/alice", "@type": "Person", "name": "Alice" }

Turtle

@prefix foaf: <http://xmlns.com/foaf/0.1/> . <http://example.org/alice> a foaf:Person ; foaf:name "Alice" .

9. 실전 체크리스트

JSON-LD 작성 체크리스트

  • @context는 문서 최상위에 포함했나?
  • @type으로 리소스 타입을 명시했나?
  • URI가 필요한 리소스에 @id를 추가했나?
  • Schema.org 타입을 최대한 활용했나?
  • JSON 문법이 유효한가? (쉼표, 중괄호 확인)
  • Google Rich Results Test로 검증했나?

10. 핵심 정리

JSON-LD를 배우고 나면

  • JSON 형식으로 RDF 데이터를 표현할 수 있습니다
  • @context, @id, @type, @graph의 역할을 이해합니다
  • Schema.org 구조화 데이터를 웹페이지에 추가할 수 있습니다
  • 웹 API를 시맨틱하게 설계할 수 있습니다
  • 도서관 메타데이터를 JSON-LD로 표현할 수 있습니다
  • SEO 최적화를 위한 구조화 데이터를 작성할 수 있습니다

핵심 키워드

키워드 의미 예시
@context 용어의 의미 정의 "@context": "https://schema.org"
@id 리소스 URI "@id": "http://example.org/alice"
@type 리소스 타입 (클래스) "@type": "Person"
@graph 여러 리소스 그룹화 "@graph": [...]
@language 언어 지정 "@language": "ko"
@value 리터럴 값 {"@value": "30", "@type": "xsd:integer"}

마무리

JSON-LD는 웹 개발자를 위한 RDF입니다. 익숙한 JSON 문법을 사용하면서도 시맨틱 웹의 강력함을 그대로 활용할 수 있습니다.

특히 Schema.org와의 통합으로 검색엔진 최적화(SEO)에도 큰 도움이 됩니다. 도서관 웹사이트나 API를 구축할 때 JSON-LD를 활용하면 데이터의 발견성과 재사용성을 크게 높일 수 있습니다.
