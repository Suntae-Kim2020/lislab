N-Triples와 N3: 단순함과 강력함

N-Triples와 N3란?

N-Triples는 RDF를 표현하는 가장 단순한 형식입니다. 한 줄에 하나의 Triple만 표현하여 처리가 매우 쉽습니다.

N3 (Notation3)는 Turtle의 전신으로, Turtle보다 더 강력한 기능(변수, 추론 규칙)을 제공합니다.

  • N-Triples: 2014년 W3C 권고안, 가장 단순한 RDF 형식
  • N3: 1998년 Tim Berners-Lee 개발, 추론 기능 포함
  • 특징: N-Triples는 극도로 단순, N3는 확장 기능 제공
  • 활용: 대용량 데이터 처리, 스트리밍, 추론 규칙

1. N-Triples: 가장 단순한 RDF

1.1 N-Triples의 철학

단순함의 미학

N-Triples는 "한 줄에 하나의 Triple만"이라는 원칙을 따릅니다.

  • Prefix 없음 (모든 URI 전체 표기)
  • 줄임말 없음 (세미콜론, 쉼표 사용 안함)
  • 각 Triple은 독립적
  • 파싱이 매우 간단함

1.2 N-Triples 기본 문법

N-Triples의 규칙

# 기본 형태: <주어> <술어> <목적어> . <http://example.org/person/alice> <http://xmlns.com/foaf/0.1/name> "Alice" . <http://example.org/person/alice> <http://xmlns.com/foaf/0.1/age> "30"^^<http://www.w3.org/2001/XMLSchema#integer> . <http://example.org/person/alice> <http://xmlns.com/foaf/0.1/knows> <http://example.org/person/bob> .

특징:

  • 모든 URI는 < >로 감쌈
  • 문자열은 " "로 감쌈
  • 각 줄은 .으로 끝남
  • 한 줄에 정확히 하나의 Triple

1.3 Turtle과 비교

Turtle (간결함)

@prefix foaf: <http://xmlns.com/foaf/0.1/> . @prefix ex: <http://example.org/person/> . ex:alice foaf:name "Alice" ; foaf:age 30 ; foaf:knows ex:bob .

사람이 읽기 좋음

N-Triples (단순함)

<http://example.org/person/alice> <http://xmlns.com/foaf/0.1/name> "Alice" . <http://example.org/person/alice> <http://xmlns.com/foaf/0.1/age> "30"^^<http://www.w3.org/2001/XMLSchema#integer> . <http://example.org/person/alice> <http://xmlns.com/foaf/0.1/knows> <http://example.org/person/bob> .

기계가 처리하기 좋음

1.4 N-Triples의 장점

왜 N-Triples를 사용하나?

  • 스트리밍 처리: 한 줄씩 읽으며 바로 처리 가능
  • 병렬 처리: 각 줄이 독립적이라 병렬화 쉬움
  • 대용량 데이터: 메모리에 전체 파일을 올릴 필요 없음
  • 간단한 파싱: 정규 표현식만으로 파싱 가능
  • 디버깅 용이: 특정 Triple만 찾기 쉬움 (grep 사용)
  • 결합 쉬움: 여러 파일을 단순히 합치기만 하면 됨

1.5 실전 예시: 도서 데이터

N-Triples로 표현한 책 정보

<http://library.example.org/book/12345> <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <http://purl.org/ontology/bibo/Book> . <http://library.example.org/book/12345> <http://purl.org/dc/elements/1.1/title> "도서관 정보학 입문"@ko . <http://library.example.org/book/12345> <http://purl.org/dc/elements/1.1/creator> <http://library.example.org/author/kim> . <http://library.example.org/book/12345> <http://purl.org/dc/elements/1.1/publisher> "한국출판사" . <http://library.example.org/book/12345> <http://purl.org/dc/elements/1.1/date> "2023-03-15"^^<http://www.w3.org/2001/XMLSchema#date> . <http://library.example.org/book/12345> <http://purl.org/ontology/bibo/isbn> "978-89-1234-567-8" . <http://library.example.org/author/kim> <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <http://xmlns.com/foaf/0.1/Person> . <http://library.example.org/author/kim> <http://xmlns.com/foaf/0.1/name> "김철수"@ko .

2. N3: 추론 가능한 RDF

2.1 N3의 특징

Turtle의 선배, N3

N3 (Notation3)는 Tim Berners-Lee가 1998년에 만든 RDF 직렬화 형식입니다.

Turtle은 N3의 부분집합으로, N3의 간단한 버전이라고 볼 수 있습니다.

  • Turtle 문법을 모두 포함
  • 변수 사용 가능 (?x, ?y)
  • 추론 규칙 표현 가능 (=>)
  • 온톨로지 추론에 사용

2.2 N3의 기본 문법

N3는 Turtle + α

N3의 기본 문법은 Turtle과 동일합니다:

@prefix foaf: <http://xmlns.com/foaf/0.1/> . @prefix ex: <http://example.org/> . ex:alice a foaf:Person ; foaf:name "Alice" ; foaf:age 30 .

이 코드는 Turtle과 N3 모두에서 유효합니다.

2.3 N3의 확장 기능: 변수

변수 사용하기

N3에서는 ?변수명 형태로 변수를 선언할 수 있습니다:

@prefix foaf: <http://xmlns.com/foaf/0.1/> . # "어떤 사람이 30살이면"이라는 패턴 ?person a foaf:Person . ?person foaf:age 30 .

이는 추론 규칙에서 활용됩니다.

2.4 N3의 확장 기능: 추론 규칙

추론 규칙 (Implication)

N3의 가장 강력한 기능은 => 연산자를 사용한 추론입니다:

@prefix foaf: <http://xmlns.com/foaf/0.1/> . @prefix ex: <http://example.org/> . # 규칙: X가 Y를 알고, Y가 Z를 알면, X도 Z를 알 가능성이 있다 { ?x foaf:knows ?y . ?y foaf:knows ?z . } => { ?x ex:mayKnow ?z . } .

이 규칙은 "친구의 친구는 잠재적 친구"라는 추론을 표현합니다.

실전 예시: 도서관 규칙

@prefix bibo: <http://purl.org/ontology/bibo/> . @prefix dc: <http://purl.org/dc/elements/1.1/> . @prefix ex: <http://example.org/> . # 규칙: 책의 저자가 교수이면, 그 책은 학술서적이다 { ?book a bibo:Book . ?book dc:creator ?author . ?author a ex:Professor . } => { ?book a ex:AcademicBook . } .

3. 세 형식의 차이점

형식 복잡도 기능 주요 용도
N-Triples 매우 단순 기본 Triple만 대용량 데이터, 스트리밍
Turtle 중간 Prefix, 축약 문법 사람이 읽고 쓰기
N3 복잡 변수, 추론 규칙 추론 엔진, 규칙 기반 시스템

3.1 동일한 데이터, 세 가지 표현

N-Triples 버전

<http://example.org/alice> <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <http://xmlns.com/foaf/0.1/Person> . <http://example.org/alice> <http://xmlns.com/foaf/0.1/name> "Alice" . <http://example.org/alice> <http://xmlns.com/foaf/0.1/knows> <http://example.org/bob> .

Turtle 버전

@prefix foaf: <http://xmlns.com/foaf/0.1/> . @prefix ex: <http://example.org/> . ex:alice a foaf:Person ; foaf:name "Alice" ; foaf:knows ex:bob .

N3 버전 (추론 규칙 추가)

@prefix foaf: <http://xmlns.com/foaf/0.1/> . @prefix ex: <http://example.org/> . ex:alice a foaf:Person ; foaf:name "Alice" ; foaf:knows ex:bob . # N3만의 추가 기능: 추론 규칙 { ?x foaf:knows ?y } => { ?y foaf:knows ?x } .

4. 언제 어떤 형식을 사용할까?

N-Triples를 사용할 때

  • 대용량 데이터를 처리할 때 (GB, TB 단위)
  • 데이터를 스트리밍 방식으로 처리할 때
  • 단순한 파서를 만들어야 할 때
  • 병렬 처리가 필요할 때
  • 데이터 덤프를 저장할 때
  • 줄 단위로 데이터를 검색할 때 (grep, awk)

Turtle을 사용할 때

  • 사람이 직접 RDF 파일을 작성할 때
  • 온톨로지를 설계할 때
  • 메타데이터를 문서화할 때
  • 소규모~중규모 데이터셋일 때
  • 가독성이 중요할 때

N3를 사용할 때

  • 추론 규칙을 표현해야 할 때
  • 규칙 기반 시스템을 구축할 때
  • 온톨로지 추론이 필요할 때
  • 복잡한 논리를 표현할 때
  • N3 추론 엔진 (cwm, EYE)을 사용할 때

5. 실무 활용 사례

5.1 대용량 데이터 처리 with N-Triples

셸 스크립트로 N-Triples 처리

# 특정 술어를 가진 Triple만 추출 grep "foaf/0.1/name" data.nt # Alice가 주어인 모든 Triple 찾기 grep "<http://example.org/alice>" data.nt # Triple 개수 세기 wc -l data.nt # 두 파일 병합 cat file1.nt file2.nt > merged.nt

N-Triples의 단순함 덕분에 Unix 도구로 쉽게 처리할 수 있습니다.

5.2 Python으로 N-Triples 처리

스트리밍 방식으로 대용량 파일 처리

from rdflib import Graph # 대용량 N-Triples 파일을 한 줄씩 처리 g = Graph() with open('large_data.nt', 'r') as f: for line in f: # 한 줄씩 파싱하여 처리 g.parse(data=line, format='nt') # 필요한 처리 수행... if len(g) > 10000: # 10,000개씩 배치 처리 process_batch(g) g = Graph() # 초기화

5.3 N3 추론 예시

도서관 추천 시스템

@prefix bibo: <http://purl.org/ontology/bibo/> . @prefix dc: <http://purl.org/dc/elements/1.1/> . @prefix ex: <http://example.org/> . # 규칙 1: 같은 저자의 책은 유사한 책이다 { ?book1 dc:creator ?author . ?book2 dc:creator ?author . ?book1 a bibo:Book . ?book2 a bibo:Book . } => { ?book1 ex:similarTo ?book2 . } . # 규칙 2: 유사한 책을 빌린 사용자에게 추천 { ?user ex:borrowed ?book1 . ?book1 ex:similarTo ?book2 . } => { ?user ex:recommended ?book2 . } .

6. 파일 크기 비교

같은 데이터, 다른 크기

10,000개 Triple을 표현할 때 예상 파일 크기:

형식 파일 크기 비고
N-Triples ~2.5 MB 모든 URI 전체 표기
Turtle ~1.2 MB Prefix 사용으로 축약
N3 ~1.2 MB Turtle과 유사 (규칙 제외)
XML/RDF ~4.0 MB XML 태그로 인해 가장 큼
JSON-LD ~1.8 MB JSON 문법 오버헤드

N-Triples는 압축을 사용하면 파일 크기를 크게 줄일 수 있습니다 (.nt.gz).

7. 도구 및 라이브러리

N-Triples/N3 처리 도구

  • rdflib (Python): N-Triples, Turtle, N3 모두 지원
  • Apache Jena (Java): 강력한 RDF 처리 라이브러리
  • rapper (Raptor): RDF 형식 변환 도구
  • cwm: N3 추론 엔진 (Tim Berners-Lee 개발)
  • EYE: 고성능 N3 추론 엔진
  • riot (Jena): 커맨드라인 RDF 도구

형식 변환 예시

# Turtle을 N-Triples로 변환 rapper -i turtle -o ntriples input.ttl > output.nt # N-Triples를 Turtle로 변환 rapper -i ntriples -o turtle input.nt > output.ttl # Jena riot 사용 riot --output=NT input.ttl > output.nt

8. N-Triples vs N-Quads

N-Quads: N-Triples + Named Graph

N-Quads는 N-Triples에 네 번째 요소(그래프 이름)를 추가한 형식입니다:

# N-Triples (3개 요소) <http://example.org/alice> <http://xmlns.com/foaf/0.1/name> "Alice" . # N-Quads (4개 요소: 주어, 술어, 목적어, 그래프) <http://example.org/alice> <http://xmlns.com/foaf/0.1/name> "Alice" <http://example.org/graph1> .

여러 그래프를 하나의 파일에 저장할 때 유용합니다.

9. 핵심 정리

N-Triples와 N3를 배우고 나면

  • 대용량 RDF 데이터를 효율적으로 처리할 수 있습니다
  • 스트리밍 방식으로 데이터를 다룰 수 있습니다
  • Unix 도구(grep, awk)로 RDF를 다룰 수 있습니다
  • 추론 규칙을 N3로 표현할 수 있습니다
  • 상황에 맞는 RDF 형식을 선택할 수 있습니다

형식 선택 가이드

상황 추천 형식 이유
대용량 데이터 (10GB+) N-Triples 스트리밍, 병렬 처리
사람이 직접 편집 Turtle 가독성, 간결함
추론 규칙 필요 N3 변수, 추론 지원
웹 API JSON-LD JSON 호환성
시스템 간 교환 XML/RDF 광범위한 지원
온톨로지 설계 Turtle 사람 친화적

마무리

N-Triples는 단순함으로 대용량 데이터 처리에 최적화되어 있고, N3는 강력한 추론 기능으로 규칙 기반 시스템에 유용합니다.

Turtle을 배웠다면 N-Triples는 그 단순화 버전, N3는 확장 버전으로 쉽게 이해할 수 있습니다. 각 형식의 장단점을 이해하고 상황에 맞게 선택하는 것이 중요합니다.

도서관에서 대용량 메타데이터를 다루거나, 추천 시스템을 구축할 때 이 두 형식을 활용하면 효율적인 시스템을 만들 수 있습니다.
